# 0.2.4 — 2026-07-16

## Added

- **`offset` pagination:** `met_search_collections` accepts an `offset` input (default 0) and returns `nextOffset`/`remaining` output fields — each page re-runs the upstream search and slices it locally, so a broad, unfiltered query carries the same timeout risk on every page. ([#9](https://github.com/cyanheads/met-museum-mcp-server/issues/9))
- **`departmentId` validation:** an unrecognized `departmentId` now throws `invalid_department` (recovery points to `met_list_departments`) instead of silently falling through to a `no_results` response. ([#7](https://github.com/cyanheads/met-museum-mcp-server/issues/7))

## Fixed

- **`invalid_date_range`:** both throw sites now pass `ctx.recoveryFor('invalid_date_range')`, so the declared recovery hint reaches `data.recovery.hint` and the `content[]` text — previously silent, unlike the neighboring `no_results` path. ([#6](https://github.com/cyanheads/met-museum-mcp-server/issues/6))
- **Broad-query timeouts:** a deterministic upstream timeout on an unfiltered `met_search_collections` query now fails fast as a non-retryable `search_timeout` (~10s) instead of retrying the same timeout to exhaustion (~48s), with recovery guidance to narrow the query or add filters. ([#11](https://github.com/cyanheads/met-museum-mcp-server/issues/11))
