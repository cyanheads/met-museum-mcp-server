# 0.2.0 — 2026-06-08

## Added

- **`isOnView` filter** on `met_search_collections` ([#3](https://github.com/cyanheads/met-museum-mcp-server/issues/3)) — optional boolean; when `true`, restricts results to objects currently on display in a Met gallery. Live-verified: narrows Rembrandt results 1096→117. Maps directly to the `isOnView` query param on `GET /public/collection/v1/search`.

## Changed

- **`met_search` renamed to `met_search_collections`** ([#4](https://github.com/cyanheads/met-museum-mcp-server/issues/4)) — **BREAKING.** Clients calling `met_search` must update to `met_search_collections`. No change to inputs, outputs, or behavior beyond the name. The new name matches the existing `met_<verb>_<noun>` pattern (`met_list_departments`, `met_get_object`) and gives callers clearer scope context.

  **Migration:** replace `met_search` with `met_search_collections` in all tool call sites.

- **`@types/node`** `25.9.1 → 25.9.2` (dev dep, lock-resolved)
