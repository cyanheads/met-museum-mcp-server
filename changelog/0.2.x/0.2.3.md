# 0.2.3 — 2026-07-16

## Changed

- **Framework:** `@cyanheads/mcp-ts-core` adopted at `^0.10.14`.
- **Supply-chain hardening:** `bunfig.toml` adds a Socket install scanner (`[install.security]`, pre-install malware/typosquat/CVE scanning) and a 3-day `minimumReleaseAge` hold on newly published package versions, excluding `@cyanheads/mcp-ts-core` for same-day framework adoption.
- **`scripts/devcheck.ts`**: the outdated-dependency check now treats packages held back by `minimumReleaseAge` (`bun outdated`'s `*` marker) as expected, not a failure.
- **TypeScript 6:** upgraded from `^5.9.3` to `^6.0.3`.
- **Docker:** `oven/bun:1.3` pinned to `oven/bun:1.3.14`, BuildKit cache mounts added to the `bun install` layers, and the build-stage install now runs with `--ignore-scripts`.
- **Repo policy files added:** `.github/SECURITY.md` (vulnerability reporting), `.gitattributes` (LF normalization, binary-asset markers, `linguist-generated` tags on generated docs), `.github/FUNDING.yml`.
- `manifest.json` gains a `license` field; `package.json` `author` now links the author's own site instead of the repo README.
- 12 vendored `skills/<name>/SKILL.md` files (plus 4 reference docs) re-synced to their mcp-ts-core 0.10.14 bodies.

## Fixed

- **`met_get_object`** ([#5](https://github.com/cyanheads/met-museum-mcp-server/issues/5)): `objects[]`/`failed[]` now preserve the caller's `objectIDs` order — an index-addressed cursor writes each result to its input position instead of a completion-ordered push into a shared array.
