# 0.2.5 — 2026-07-16

## Changed

- **Tool descriptions:** `met_search_collections`, `met_get_object`, and `met_list_departments` descriptions and field `.describe()` text rewritten to descriptive voice — dropped imperative coaching (`Always chain…`, `Use before…`, `prefer using…`) and internal mechanics (`fetches run in parallel (concurrency-limited)`, per-page offset slicing). ([#8](https://github.com/cyanheads/met-museum-mcp-server/issues/8))
- **Stale claims corrected:** `met_list_departments` no longer states the department list is "fetched live on each call" (a cache exists), and `docs/design.md` replaces "No pagination on search results" with the server's `offset`/`limit` layering over the Met `/search` single-response set.

## Fixed

- **`met_get_object` content completeness:** `format()` renders every field unconditionally, using an em-dash (`—`) placeholder for empty/null/false values (new `orDash` helper) and `**Failed fetches:** none` for an empty `failed[]`. A sparse object (e.g. object 1) no longer drops ~11 fields from `content[]` that `structuredContent` already carries. ([#10](https://github.com/cyanheads/met-museum-mcp-server/issues/10))
- **`met_search_collections` completion marker:** a non-truncated page renders `(complete)` instead of nothing, surfacing the `truncated: false` state in `content[]`.
