# 0.2.2 — 2026-06-20

## Changed

- **`devcheck.config.json`** — added `packaging.pluginManifests: true`, enabling `lint:packaging` check 10: `.claude-plugin/plugin.json`, `.codex-plugin/plugin.json`, and `.codex-plugin/mcp.json` are now validated for non-empty descriptions, unscoped machine-name display fields, and a full-name `npx -y` install arg. The "Packaging" gate now also runs when a plugin manifest is present, not only when `manifest.json` exists. ([cyanheads/mcp-ts-core#240](https://github.com/cyanheads/mcp-ts-core/issues/240))
- **`scripts/check-dependency-specifiers.ts`** — new vendored devcheck step (`Dependency Specifiers`) that hard-fails on floating specifiers (`latest`, `*`, pre-release dist-tags) in `package.json` dependency sections and `bun.lock` workspace specifier maps. ([cyanheads/mcp-ts-core#246](https://github.com/cyanheads/mcp-ts-core/issues/246))
- **Vendored devcheck scripts re-synced** to mcp-ts-core 0.10.7 — `build-changelog.ts`, `check-framework-antipatterns.ts`, `check-skill-versions.ts`, and `devcheck.ts` gain fresh-scaffold guards (changelog `--check` exits cleanly on a template-only tree; git-dependent checks skip when `.git` is absent; `check-skill-versions` skips a `SKILL.md` deleted from the worktree). `lint-packaging.ts` carries the new plugin-manifest checks. ([cyanheads/mcp-ts-core#237](https://github.com/cyanheads/mcp-ts-core/issues/237), [cyanheads/mcp-ts-core#242](https://github.com/cyanheads/mcp-ts-core/issues/242), [cyanheads/mcp-ts-core#243](https://github.com/cyanheads/mcp-ts-core/issues/243))
- **Fourteen vendored `skills/<name>/SKILL.md` files re-synced** to their mcp-ts-core 0.10.7 versions — propagates body drift on seven skills bumped to clear the version-gated maintenance sync (`api-auth`, `api-errors`, `api-services`, `api-telemetry`, `field-test`, `report-issue-local`, `tool-defs-analysis`), plus refreshed `git-wrapup` v1.4 and `orchestrations` v1.4 guidance. ([cyanheads/mcp-ts-core#238](https://github.com/cyanheads/mcp-ts-core/issues/238))
