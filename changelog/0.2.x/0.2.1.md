# 0.2.1 — 2026-06-12

## Added

- **`name` and `title`** set explicitly on `createApp()` in `src/index.ts` — both pinned to `met-museum-mcp-server` so the hyphenated repo name is the display identity on every client surface, not the npm-scoped package name.
- **Docker `HEALTHCHECK`** in the `Dockerfile` — polls `/healthz` every 30s via `bun -e fetch`, no `curl`/`wget` dependency.
- **`org.opencontainers.image.version` OCI label** stamped from the `APP_VERSION` build arg.

## Changed

- **`invalid_date_range` error** on `met_search_collections` now classifies as `JsonRpcErrorCode.ValidationError` (was `InvalidParams`) — a cross-field input-consistency failure is a domain validation error, not a malformed-params protocol error.
- **`bundle` script** now runs `scripts/clean-mcpb.ts` after packing — prunes dev dependencies and strips dependency-shipped agent docs (`skills/`, `.claude/`, `.agents/`, `SKILL.md` under `node_modules/**`) that root-anchored `.mcpbignore` patterns can't reach.
- **`.mcpbignore`** patterns anchored to the bundle root (`/skills/`, `/Dockerfile`, …) so they match only top-level entries, not nested `node_modules` paths.

## Fixed

- **Dependency bumps:**
  - `@cyanheads/mcp-ts-core` `^0.9.19 → ^0.10.6`
  - `@biomejs/biome` `^2.4.7 → ^2.4.16` (dev)
  - `@types/node` `^25.9.2 → ^25.9.3` (dev)
  - `tsc-alias` `^1.8.16 → ^1.8.17` (dev)
